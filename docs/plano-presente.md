# Plano de entrega surpresa (presente) — DuoGestão

## Contexto
Este projeto será entregue como presente. A utilizadora final (tia) ainda não participou do levantamento de requisitos.

## Estratégia de execução sem entrevista inicial
Como não há descoberta direta no início, a abordagem será:
1. Criar um MVP guiado por padrões de micro negócio de retalho + encomendas.
2. Priorizar fluxos universais (venda, stock, encomenda, caixa diário, resumo semanal).
3. Entregar uma versão "demo real" com dados fictícios semelhantes à rotina dela.
4. Recolher feedback apenas após a primeira apresentação.

## Escopo do MVP para máxima utilidade imediata
- Dashboard com indicadores essenciais (dia/semana/mês)
- Gestão de produtos e stock
- Registo de encomendas de sopa
- Lançamento de entradas e saídas
- Resumo semanal automático
- Emissão simples de recibos

## Decisões para aumentar chance de aceitação
- Interface limpa, sem excesso de campos obrigatórios
- Linguagem simples nos botões e mensagens
- Fluxos rápidos para tarefas repetidas
- Relatórios fáceis de entender por não técnicos

## Roteiro de entrega em 3 marcos

### Marco 1 — Base funcional (Semana 1)
- Estrutura da app
- Login
- Layout profissional
- Módulo de produtos com stock

### Marco 2 — Operação diária (Semana 2)
- Encomendas de sopa
- Entradas e saídas financeiras
- Caixa diário consolidado

### Marco 3 — Valor percebido (Semana 3)
- Relatório semanal
- Recibo/fatura simplificada
- Ajustes visuais finos e onboarding

## Checklist de apresentação (dia do presente)
- Conta principal criada
- Dados de exemplo preenchidos (7 a 14 dias simulados)
- PDF de resumo semanal pronto para mostrar
- 3 fluxos ensaiados: venda, encomenda, fechamento semanal

## Pós-apresentação (1ª semana de uso real)
- Recolher dificuldades de uso
- Ajustar labels, campos e atalhos
- Adicionar melhorias rápidas de alto impacto
